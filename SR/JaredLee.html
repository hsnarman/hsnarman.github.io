The Project

During the spring semester of 2018, I am working with my mentor Dr. Husnu Narman and fellow group member Geanina to develop a tool that uses various API's to analyze an IP addresses reputability. We have the developed the capability for this, however, we are wanting to extend it to be able to use machine learning to identify an IP Address and determine its reputability. The demand for this is high, as its impossible for blacklists to locate every single malicious domain. On this web page, I will be recording a weekly blog of my research as well as posting the resources I have been using to develop this project.

Reports

Week One

During the first week, I began to familiarize myself with the project description as well as begin to learn machine learning. The web application has already been created by my group member using PHP, however, it still needs a lot of work and needs a machine learning implementation.

Week Two

On the second week, I implemented a small python project that utilized supervised learning with logistic regression. It has an excel spreadsheet of over 300,000 URL's and classified them as good or bad. It managed to get an accuracy rating of 96% when using 20% of the data set for training, however, using only URL information is not enough to accurately classify URL's and determine if they are malicious.

Week Three

This week I developed a python project that will run daily API scans and dump the results to a SQL Lite3 database. I also created a serializer for my machine learning implementation so we can begin implementing it on the web application. This is still an extremely rudimentary form of what a good classifier should look at by the end of this project, however, it will give us an idea of the requirements necessary for it to perform on the application.

Week Four

During the fourth week, I implemented the python project with the front end of the PHP application. It populates a HTML table and reads the SQLite database file. I also implemented the machine learning python file with the project so we can demo this for our presentation.

Week Five

On the fifth week, we presented our project to some of the staff members at Marshall University as well as some other students who are participating in research projects. The presentation went okay, however, a lot more work is required for the machine learning to be applicable to a real application.

Week Six

On the sixth week, we were asked from one of the professors at Marshall University to re-design the website. A system diagram was prepared as well as a wire diagram for the new design of the web application.

Week Seven

On the seventh week, some adjustment was made to the diagram and we are currently still waiting approval for the design before implementing it.

Week Eight

On this week, we discussed some of the challenges in preparing a dynamic data-set. Using selenium, a crawler was made to gather IP's from mass black-lists. However, more features are required for IP's, most important, geo information.

Week Nine

This week we used an API to obtain geo location information on ip address entries. Ip addresses were crawled on websites using selenium.

Week Ten

This week we developed papers for the project to apply for the West Virginia Undergraduate day at the capitol.

Week Eleven

A New design has been made and is being implemented.

Week Twelve

Instead of a black/blue theme, we opted for a clean white, modern background. The application in its prototype has been built. We are planning on meeting with our mentor to discuss its progress next week.
